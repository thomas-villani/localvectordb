Inspector Disabled

The LocalVectorDB Inspector interface is currently disabled. To use the Inspector, you need to enable it in your server configuration.

ℹ️ What is the Inspector?

The LocalVectorDB Inspector is a web-based interface that allows you to:

  • Browse and search your vector databases
  • Visualize document embeddings and clusters
  • Test and optimize search queries
  • Monitor system performance and health
  • Manage API keys and configuration

🔧 How to Enable the Inspector

  1. Open your LocalVectorDB configuration file (usually .lvdb-config.toml or .lvdb-config.json)
  2. Add or update the inspector setting in the [server] section:
    # TOML Configuration [server] inspector_enabled = true
    Or for JSON configuration:
    // JSON Configuration { "server": { "inspector_enabled": true } }
  3. Save the configuration file
  4. Restart your LocalVectorDB server:
    # Restart the server lvdb serve
  5. The Inspector will be available at /inspector

⚠️ Security Consideration

The Inspector provides administrative access to your LocalVectorDB system. If you're running in production, make sure to secure it with API key authentication and consider restricting network access.
